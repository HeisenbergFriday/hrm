# 钉钉一体化人事后台前端设计

## 1. 技术栈

- React 18+
- Vite 6+
- TypeScript 5+
- Ant Design 5+
- React Query 5+
- Zustand 4+

## 2. 页面结构

### 2.1 整体布局

- **顶部导航栏**：系统标题、用户信息、退出按钮
- **左侧菜单栏**：功能模块导航
- **主内容区**：根据选择的功能模块显示对应内容
- **底部状态栏**：系统状态、版本信息

### 2.2 页面列表

| 页面名称 | 路径 | 功能描述 |
|---------|------|---------|
| 登录页面 | `/login` | 用户登录 |
| 首页 | `/` | 系统概览、数据统计 |
| 组织架构 | `/organization` | 部门树、员工列表、员工详情 |
| 考勤管理 | `/attendance` | 考勤记录查询、考勤统计 |
| 审批管理 | `/approval` | 审批列表、审批详情 |
| 权限管理 | `/permission` | 角色管理、权限设置 |
| 操作日志 | `/log` | 日志查询、日志详情 |
| 系统设置 | `/setting` | 系统配置、同步设置 |

## 3. 组件设计

### 3.1 通用组件

1. **Layout**：整体布局组件
2. **Header**：顶部导航栏组件
3. **Sidebar**：左侧菜单栏组件
4. **Footer**：底部状态栏组件
5. **PageHeader**：页面头部组件
6. **Table**：表格组件（支持分页、排序、筛选）
7. **Form**：表单组件（支持验证）
8. **Modal**：对话框组件
9. **Drawer**：抽屉组件
10. **Message**：消息提示组件
11. **Notification**：通知组件
12. **Loading**：加载状态组件
13. **Empty**：空状态组件
14. **ErrorBoundary**：错误边界组件

### 3.2 业务组件

1. **DepartmentTree**：部门树组件
2. **EmployeeList**：员工列表组件
3. **EmployeeDetail**：员工详情组件
4. **AttendanceTable**：考勤记录表格组件
5. **AttendanceStatistics**：考勤统计组件
6. **ApprovalList**：审批列表组件
7. **ApprovalDetail**：审批详情组件
8. **RoleList**：角色列表组件
9. **PermissionSetting**：权限设置组件
10. **LogList**：操作日志列表组件
11. **LogDetail**：操作日志详情组件
12. **SyncStatus**：同步状态组件

## 4. 状态管理

使用 Zustand 管理全局状态，主要包括：

1. **AuthStore**：认证状态管理
   - 用户信息
   - 登录状态
   - 权限信息

2. **OrganizationStore**：组织架构状态管理
   - 部门树数据
   - 员工列表数据
   - 选中的部门和员工

3. **AttendanceStore**：考勤状态管理
   - 考勤记录数据
   - 考勤统计数据
   - 查询条件

4. **ApprovalStore**：审批状态管理
   - 审批列表数据
   - 审批详情数据
   - 查询条件

5. **PermissionStore**：权限状态管理
   - 角色列表数据
   - 权限设置数据

6. **LogStore**：操作日志状态管理
   - 日志列表数据
   - 日志详情数据
   - 查询条件

7. **SettingStore**：系统设置状态管理
   - 系统配置
   - 同步设置

## 5. API 调用

使用 React Query 处理 API 调用和缓存，主要包括：

1. **Auth API**：登录、登出、获取用户信息
2. **Organization API**：获取部门树、获取员工列表、获取员工详情
3. **Attendance API**：获取考勤记录、获取考勤统计
4. **Approval API**：获取审批列表、获取审批详情
5. **Permission API**：获取角色列表、获取权限设置、更新权限
6. **Log API**：获取操作日志、获取日志详情
7. **Sync API**：同步部门、同步员工、同步考勤、同步审批

## 6. 页面状态处理

每个页面都需要处理以下状态：

1. **Loading**：数据加载中状态
2. **Empty**：数据为空状态
3. **Error**：数据加载错误状态
4. **Retry**：错误重试功能

## 7. 响应式设计

- **桌面端**：完整布局，左侧菜单栏展开
- **平板端**：左侧菜单栏可折叠
- **移动端**：底部导航栏，顶部标题栏

## 8. 主题设计

- **主色调**：蓝色系（#1890ff）
- **辅助色**：灰色系、绿色系、红色系
- **字体**：系统默认字体
- **布局**：基于 Ant Design 的响应式布局

## 9. 性能优化

1. **组件懒加载**：使用 React.lazy 和 Suspense 实现组件懒加载
2. **数据缓存**：使用 React Query 缓存 API 响应数据
3. **虚拟列表**：对于长列表使用虚拟滚动
4. **代码分割**：使用 Vite 的代码分割功能
5. **图片优化**：使用适当的图片格式和大小

## 10. 无障碍设计

- 支持键盘导航
- 支持屏幕阅读器
- 符合 WCAG 2.1 标准

## 11. 国际化

- 支持多语言切换
- 使用 i18n 库管理翻译
